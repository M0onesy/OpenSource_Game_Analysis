开源，还是闭源？

大模型公司战略选择的博弈论分析

OpenAI、Anthropic 坚持闭源，仅提供 API 而不公开模型权重；Meta、DeepSeek、阿里则公开权重。这一分化是短期策略，还是稳定的均衡结构？本文尝试用博弈论给出可推导、可检验的解释。

01研究问题

本文要回答的三个问题

1
为何出现策略分化？同为大模型厂商，有的选择开源，有的坚持闭源。
2
这一分化是否稳定？是过渡状态，还是任一方单方面改变都会受损的均衡。
3
能否事前推导而非事后归因？即以博弈论给出可验证的预测。
本文的判断是：一家闭源垄断高端、其余开源争夺生态的分层共存并非偶然，而是一个可被严格刻画、并能给出可检验预测的稳定均衡。

分析框架 · 路线图

全文的分析路线

本文先用一个数据驱动的支付函数刻画产业、求出基线均衡，再依次改变博弈的三个要素，从三个角度深化结论。五个模型不是彼此独立，而是对同一产业的逐层加深。

第一步 · 基线博弈

谁会开源、谁会闭源

  • 静态博弈：三方同时决策，求纳什均衡
  • 序贯博弈：引入先后顺序，求子博弈完美均衡
  • 两者结论一致 → 均衡为 (闭, 开, 开)
第二步 · 三项扩展

放入更真实的设定

  • 演化加入监管者，看群体的长期比例
  • 信号信息不完全，看透明度的作用
  • 重复多期博弈，看高价能否维持
第三步 · 综合

得到什么结论

  • 开源已成不可逆力量，与闭源前沿长期并存
  • 每个模型都给出可与现实对照的数值
  • 据此对未来一至两年作出预测

先回答「会怎样」（基线均衡），再回答「为什么稳、何时会变」（三项扩展）——这条主线贯穿后续每一页。

05方法核心

支付由数据推导，而非主观设定

若直接假设各方支付，便陷入循环论证。本文先构建将策略映射为效用的支付函数，再以真实产业数据将其实例化，使每一个支付值都可逐项追溯。

Ui = RAPI·𝟙[闭源] + α·E(nO)·𝟙[开源] + Vi − κ·(ci/cref)·m

某厂商的效用，等于 API 货币化收入（仅闭源时）、开源生态价值（仅开源时）与品牌价值之和，减去成本负担。其中仅训练成本取自真实数据：GPT-4 约 7800 万美元、Llama 3.1-405B 约 1.7 亿美元、DeepSeek-V3 约 558 万美元，三者之比约 14 : 30 : 1。

① API 货币化收入

仅闭源时为正，并除以闭源厂商数 nC 以刻画竞争稀释：闭源者越多，高端市场被分摊得越薄。

② 开源生态价值

仅开源时为正，且随开源厂商增多而递减，体现开发者与心智份额的拥挤折损。

③ 品牌与战略价值

区分开源品牌与闭源品牌；DeepSeek 的开源品牌被「低成本逼近前沿」的叙事显著放大。

④ 成本负担

训练成本比直接取自真实数据；闭源独力承担研发与推理，开源由社区分担约四成，故维护乘子更低。

07均衡的唯一性

三步占优锁定唯一均衡

无需穷举：仅借助严格占优关系，三步消去全部劣势策略，即得唯一均衡。

第一步 · DeepSeek
无论 OpenAI 与 Meta 如何选择，DeepSeek 开源的支付始终更高——其闭源难获 API 收入，开源几乎不损失收入却能换取声誉与人才。消去其闭源策略。
第二步 · OpenAI
在 DeepSeek 必然开源的前提下，OpenAI 闭源严格占优：若其开源，将放弃独占高端 API 的溢价，支付由 13.1 降至 1.7。消去其开源策略。
第三步 · Meta
剩余局面下 Meta 选择开源（6.5 > 1.0）：它既无 OpenAI 的先发客户，亦无 DeepSeek 的成本优势，正面竞争将两败俱伤。三步后仅余 (闭, 开, 开)。

13趋势研判

对未来一至两年的预测

01

闭源阵营进一步分层为「前沿闭源、追赶半开、长尾全开」，gpt-oss 模式将向更多厂商扩散。

02

监管差异催生监管套利，训练与部署在不同辖区之间分离。

03

新型颠覆者（具身智能、世界模型、视频生成）将再次压低贴现因子，或引发新一轮降价。

04

开源由同质竞争转向多生态并存，Llama、Qwen、DeepSeek 各自维系社区与工具链。

05

估值与营收的领先权动态再分配；两家头部厂商的上市进程将成为下一阶段关键变量。

14方法论评估

本研究的可信度与局限

本研究的稳健性在于方法的克制与边界的明确。

01 · 支付内生化

由数据推导而非假设

支付由公式与真实成本推导，避免预设结论的循环论证。

02 · 交叉验证

多视角相互印证

静态与序贯博弈给出一致均衡，结论不依赖单一建模假设。

03 · 可证伪

给出可检验的数值

各模型均提供可与现实对照的数值：稳态比例、价格降幅、临界阈值。

04 · 边界明确

诚实标注假设

明确区分硬数据与标定量，并更正了早期的口径错误。

本研究亦不回避局限：除训练成本比为硬数据外，支付函数的其余参数、演化博弈的 reg、信号博弈的先验与似然，均为按公开证据标定的序数量，相对排序稳健而绝对数值仅服务于求解；用户侧迁移、芯片管制、循环融资等因素尚未纳入，可作后续扩展。
